1/25/20: How to use nginx to rapidly deploy other pages of your website

For first time web developers, once you have a web server up and running for the first time ever, it can be very exciting. It's important, however, to make sure you have it configured properly before you jump right into development. While there are many ways in setting up nginx (the same goes for Apache2), one method I have always found very convenient and easy to execute is enabling multiple locations in the nginx configuration file for the different areas of my website. What this means is, if I add, for example, location "/photos" to my config file, then users will then be able to navigate to "www.mywebsite.com/photos/". I have seen other people preach creating C-NAME DNS records in order to create sub-domains that point to other areas of your website (in this case, using the previous example, it would point to "photos.mywebsite.com"), but I feel that this is unnecessary work, not to mention that sub-domains were not meant to be used in that way.

In order to add these locations to your nginx configuration file, do the following: in your web server on the command line, type in
cd /etc/nginx/sites-available/ then press enter.
Then locate whichever website configuration file you wish to edit, and type
sudo nano <your-config-file-name-here> then press enter.
Locate the block of code in which the word location followed by a forward-slash appears for the first time (highlighted yellow in the image), and add underneath that declaration another block of code starting with the word location as well (refer to the example in the image). As you can see, I have added two location entries, with each location value preceeded by a forward-slash (in this case, "/memorymatch" and "/main"). Make sure you do not forget to add the forward-slash in front of your location value as well! Now, simply copy what I have done in the image, by typing the left curly brace, inside of which you will type the property "autoindex" followed by a value of "on" and a semicolon, then close it with a right curly brace. Of course, proper indentation is recommended. In case you were wondering, autoindex tells nginx to automatically locate any files named index.* and to serve up that file when users navigate to that location of your website.

Once you have finished adding all your locations, type CTRL+O to save your changes, followed by CTRL+X to exit nano. You will need to restart nginx by typing
sudo service nginx restart and pressing enter.
Now users will be able to navigate to other pages of your website by appending those location names to the end of your website name (just make sure you have index.html files in those folders on your webserver!)
